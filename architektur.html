🏛️ Architektur

Wie das System aufgebaut ist — einfach erklärt, ohne Vorkenntnisse.

1. Der Sockel — das Fundament

Wie ein Haus braucht auch eine Web-App ein Fundament. Der Sockel wird einmal gebaut und dann von allen Apps gemeinsam genutzt. Jede neue App "steht" auf diesem Fundament — ohne dass man es neu bauen muss.

🔐

Login & Nutzerverwaltung

Kunden melden sich mit Google an — kein Passwort merken, kein Formular ausfüllen. Das System erkennt automatisch, welcher Nutzer welche Apps gekauft hat.

Supabase Auth + Google OAuth
💳

Zahlung & Abos

Kunden zahlen sicher über Stripe. Ob Einzel-App (1,99 €), Flex-Paket (7,99 €/Monat) oder Unlimited (14,99 €/Monat) — alles läuft automatisch. Nach dem Kauf wird die App sofort freigeschaltet.

Stripe Checkout + Webhooks
🗄️

Datenbank

Speichert: welcher Nutzer existiert, welche Apps er gekauft hat, und alle Eingaben innerhalb der Apps. Komplett gesichert, mit Zugriffsschutz — jeder sieht nur seine eigenen Daten.

Supabase (PostgreSQL + Row Level Security)
🤖

KI-Anbindung

Jede App kann die KI ansprechen. Der Nutzer gibt etwas ein (Text, Zahlen, eine Frage) — die KI antwortet sofort mit einem nützlichen Ergebnis. Die KI läuft nie im Browser des Nutzers, sondern sicher auf dem Server.

Anthropic Claude Haiku (API)
🌐

Hosting & Geschwindigkeit

Die Seite liegt auf Cloudflare — weltweit verteilt, extrem schnell, kostenlos für den Start. Kein eigener Server nötig.

Cloudflare Pages + Next.js 14

2. Wie eine neue App entsteht

Dank des Konfigurationssystems braucht jede neue App nur eine einzige Datei — alles andere (Login, Zahlung, Layout) ist schon fertig.

📄

Schritt 1: Konfigdatei anlegen

Eine kleine Textdatei beschreibt die App: Name, Icon, Beschreibung, Preis, Kategorie. Das reicht, damit die App im Marktplatz erscheint.

src/lib/apps/meine-app.ts
↓
🧩

Schritt 2: UI-Baustein wählen

Aus fertigen Bausteinen (Chat-Fenster, Formular, Ergebnis-Karte, Tracker) wird der passende ausgewählt. Kein neues Design nötig — einfach kombinieren.

React-Komponenten aus src/components/apps/
↓
💬

Schritt 3: KI-Prompt schreiben

Ein kurzer Text erklärt der KI, was sie tun soll: "Du bist ein Einkaufslisten-Optimierer. Sortiere diese Liste nach Supermarkt-Abteilungen…" Das ist der eigentliche "Motor" der App.

Claude Haiku System-Prompt
↓
✅

Fertig — App ist live

Die App erscheint automatisch im Marktplatz, kann gekauft werden, und funktioniert sofort. Login, Zahlung und Datenspeicherung sind bereits vorhanden.

3. Der App-Generator (Phase 2)

Der nächste Schritt: nicht nur Apps bauen — sondern einen Generator bauen, der automatisch neue Apps produziert.

💡

Idee eingeben

In einem Eingabefeld (nur für Admins sichtbar) schreibt man eine App-Idee in einem Satz: "Eine App die mir hilft, meine Pflanzen zu pflegen"

↓
🤖

KI generiert alles

Claude erstellt automatisch: Konfigdatei (Name, Icon, Beschreibung, Preis), den passenden KI-Prompt für die App-Logik, und wählt den richtigen UI-Baustein aus.

Claude API → JSON-Ausgabe
↓
🚀

App ist in 30 Minuten live

Statt Stunden oder Tage zu programmieren: kurz prüfen, freigeben, deployen. Die Fabrik produziert Apps am laufenden Band.

4. Die Marketing-Maschine (Phase 3)

Jede neue App bekommt automatisch ein YouTube-Video — ohne stundenlange Videobearbeitung.

📝

Skript-Generator

Claude schreibt aus der App-Beschreibung ein vollständiges Video-Skript: Problem → Lösung → Demo → Kauflink. Immer gleiche Struktur, immer überzeugend.

↓
🖼️

Bilder & Screencasts

KI-generierte Bilder für jede Szene, kombiniert mit echten Screencast-Aufnahmen der App.

Higgsfield / Remotion
↓
🎙️

Voiceover

Text-to-Speech erstellt den deutschen Kommentar — professionelle Stimme, kein Mikrofon nötig.

ElevenLabs / Azure TTS
↓
📺

Automatischer Upload

Video wird automatisch auf YouTube hochgeladen — mit Titel, Beschreibung und Link zur App. Einmal eingerichtet, läuft die ganze Pipeline von selbst.

YouTube Data API
